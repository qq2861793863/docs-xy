### 示例 1：aiohttp 并发请求

传统 `requests` 是同步的，而 `aiohttp` 配合 `asyncio` 可以实现单线程下的高并发，极大提升抓取效率。

Python

```python
import asyncio
import aiohttp
import time

async def fetch_data(session, url):
    # 模拟异步请求
    async with session.get(url) as response:
        return await response.text()

async def main():
    urls = [f"https://httpbin.org/get?id={i}" for i in range(5)]
    
    # 使用 aiohttp 的 ClientSession 管理连接池
    async with aiohttp.ClientSession() as session:
        # 创建并发任务列表
        tasks = [asyncio.create_task(fetch_data(session, url)) for url in urls]
        # 并发执行所有任务并等待结果
        results = await asyncio.gather(*tasks)
        
        for i, res in enumerate(results):
            print(f"Task {i} completed. Length: {len(res)}")

if __name__ == '__main__':
    start_time = time.time()
    asyncio.run(main())
    print(f"总耗时: {time.time() - start_time:.2f} 秒")
```

### 示例 2：Playwright 动态渲染

当数据由复杂的 JavaScript 动态生成，且难以逆向时，使用 Playwright 模拟真实浏览器是最高效的降维打击。

Python

```python
import asyncio
from playwright.async_api import async_playwright

async def run_browser():
    async with async_playwright() as p:
        # 启动 Chromium 浏览器 (可设置 headless=False 查看界面)
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()
        
        # 伪装 User-Agent
        await page.set_extra_http_headers({"User-Agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)..."})
        
        # 访问动态网页
        await page.goto('https://quotes.toscrape.com/js/')
        
        # 等待目标元素渲染完成 (这步非常关键)
        await page.wait_for_selector('.quote')
        
        # 提取数据
        quotes = await page.locator('.quote .text').all_inner_texts()
        for quote in quotes:
            print(quote)
            
        await browser.close()

if __name__ == '__main__':
    asyncio.run(run_browser())
```

### 示例 3：JS 逆向执行（Python 调用 JS）

当我们通过断点调试找到了网页加密签名的 JS 代码后，可以直接在 Python 中运行这段 JS 来生成所需的 Token。这里使用 `PyExecJS2`。

JavaScript

```python
// 假设这是我们从网页中扣出来的加密文件 encrypt.js
function getSign(timestamp) {
    // 模拟一段混淆或加密逻辑
    var magic_number = 8192;
    return "sign_" + (timestamp * magic_number).toString(16);
}
```

Python

```python
# Python 调用端
import execjs
import time

def generate_signature():
    # 1. 加载本地扣下来的 JS 环境代码
    with open('encrypt.js', 'r', encoding='utf-8') as f:
        js_code = f.read()
    
    # 2. 编译 JS 代码
    ctx = execjs.compile(js_code)
    
    # 3. 传入参数并调用 JS 中的函数
    current_timestamp = int(time.time())
    signature = ctx.call("getSign", current_timestamp)
    
    print(f"Timestamp: {current_timestamp}")
    print(f"Generated Signature: {signature}")

if __name__ == '__main__':
    generate_signature()
```

### 示例 4：ddddocr 验证码识别

面对传统的图形验证码，除了对接打码平台，还可以使用开源的深度学习库（如带带弟弟 OCR `ddddocr`）在本地直接识别。

Python

```python
import ddddocr
import requests

def solve_captcha(image_url):
    # 1. 下载验证码图片
    response = requests.get(image_url)
    image_bytes = response.content
    
    # 2. 初始化 OCR 引擎 (底层基于神经网络)
    # show_ad=False 用于关闭启动时的控制台提示
    ocr = ddddocr.DdddOcr(show_ad=False)
    
    # 3. 识别验证码
    result = ocr.classification(image_bytes)
    print(f"识别出的验证码内容为: {result}")
    
    return result

if __name__ == '__main__':
    # 找一个返回图形验证码的测试接口
    test_url = "https://my.freenom.com/includes/domains/captchaimage.php"
    solve_captcha(test_url)
```

### 示例 5：mitmproxy App 抓包脚本

对于 App 数据，除了 Hook 内存，最常用的方法是“中间人攻击 (MITM)”。`mitmproxy` 允许我们用 Python 编写脚本，实时拦截和篡改 App 的网络请求与响应。

Python

```python
# 文件名: addon.py
# 运行方式在命令行输入: mitmdump -s addon.py
import json
from mitmproxy import http

class AppDataInterceptor:
    def response(self, flow: http.HTTPFlow):
        # 1. 过滤目标 App 的 API 接口
        target_api = "https://api.target-app.com/v1/user/list"
        
        if target_api in flow.request.pretty_url:
            # 2. 拦截并解析响应体
            try:
                response_text = flow.response.get_text()
                data = json.loads(response_text)
                
                # 3. 提取我们需要的 JSON 数据
                users = data.get("data", [])
                for user in users:
                    print(f"抓取到用户: {user['nickname']} - {user['id']}")
                    # 在这里可以将数据写入数据库 (如 MongoDB/Redis)
                    
            except json.JSONDecodeError:
                print("解析 JSON 失败")

# 注册插件
addons = [
    AppDataInterceptor()
]
```

### 阶段 5：App 抓包与中间人代理（`mitmproxy`）

`mitmproxy` 既是一个命令行抓包工具，也是一个极其强大的 Python 代理开发框架。

**安装与证书配置步骤：**

1. **安装核心工具：**

Bash

```python
# macOS 系统推荐直接用 Homebrew 安装，会自动处理全局依赖
brew install mitmproxy

# 或者通过 pip 安装
pip install mitmproxy
```

1. **启动代理服务：**

   打开终端，输入 `mitmweb`（提供网页端 UI）或 `mitmdump`（纯命令行，适合配合 Python 脚本运行）。默认会在本地 `8080` 端口启动服务。

2. **信任根证书（关键步骤）：**

   HTTPS 抓包的核心在于让设备信任 mitmproxy 生成的伪造证书。

- **电脑端：** 启动过一次 mitmproxy 后，会在用户目录下生成 `~/.mitmproxy/` 文件夹。双击打开里面的 `mitmproxy-ca-cert.pem`（或 .p12），将其安装到系统的“钥匙串”中，并手动设置为**“始终信任”**。
- **手机端（iOS/Android）：** 手机连接与电脑同一局域网的 Wi-Fi，设置 HTTP 代理为电脑的局域网 IP 和 8080 端口。然后使用手机浏览器访问 `mitm.it`，根据页面提示下载并安装证书。
