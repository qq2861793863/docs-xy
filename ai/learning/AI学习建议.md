# 一、前端学 AI 到底学什么（别走偏）

很多人一上来就：

- 学 PyTorch
- 学深度学习
- 看数学

结果 2 个月后：啥也做不出来。

👉 正确路线是分 3 层：

------

## 第一层（最重要）：AI 应用能力（必须先学）

这是**80% 前端真正需要的 AI 能力**

你要学：

### 1. LLM 基础（大模型使用）

- 什么是 prompt（提示词）
- system / user / assistant
- temperature、tokens
- 上下文（context）

👉 能力目标：

- 能控制 AI 输出风格
- 能让 AI按你格式输出（JSON、结构化）

------

### 2. Prompt 工程（核心能力）

你要会：

- few-shot（给例子）
- 角色设定
- 约束输出格式
- 防止胡说（幻觉）

👉 举例：

```text
你是一个招聘助手，请把下面内容解析成 JSON：
字段：姓名、技能、年限
```

👉 这比“会写代码调用 API”重要 10 倍

------

### 3. AI API 调用（前端 + Node）

你要会：

- 调用 AI 接口
- 流式输出（边生成边显示）
- 会话管理
- 错误处理（超时 / 重试）

👉 做一个：

- ChatGPT 页面（但带你自己的业务）

------

### 4. 结构化输出（非常关键）

让 AI 输出 JSON：

```json
{
  "title": "",
  "summary": "",
  "tags": []
}
```

👉 这是 AI 产品和“聊天工具”的本质区别

------

### 5. RAG（检索增强生成）入门

你要懂：

- embedding（向量）
- 文档切片（chunk）
- 相似度搜索
- 引用来源

👉 做一个：

- 文档问答系统（上传 PDF → 提问）

------

### 6. AI 产品思维（很多人不会）

你要考虑：

- 用户为什么用 AI
- AI 出错怎么办
- 如何给“可信度”
- 是否展示引用
- 是否允许重试

------

# 三、Python 学什么（精简清单）

注意：不是“学 Python 全部”，只学 AI 用得到的

------

## 阶段 1：Python 基础（3–5 天）

你要会：

- 变量 / 函数
- list / dict
- 文件读写
- requests（调用接口）

练习：

- 写脚本批量处理文本
- 调用 AI API 并保存结果

------

## 阶段 2：数据处理（3–5 天）

你要会：

- JSON 处理
- CSV 处理
- 文本清洗
- 批量任务

练习：

- 批量摘要 100 篇文章
- 提取关键词

------

## 阶段 3：FastAPI（重点）

这是你最该学的 Python 技术

你要会：

- 路由
- 请求/响应模型
- 参数校验
- 异步接口

做一个服务：

```bash
POST /summarize
POST /classify
POST /qa
```

------

## 阶段 4：向量 & RAG（进阶）

你要懂：

- embedding
- 向量数据库（了解即可）
- 相似度搜索

做一个：

- Python 做检索
- 前端展示结果

# 一、AI 能做什么 vs 程序员必须做什么

现在的现实是：

### AI 很擅长：

- 写 CRUD
- 搭项目骨架（Cursor / Copilot）
- 写简单逻辑
- 改 bug（简单的）
- 写 UI 页面

👉 这些能力，**已经接近“初级工程师水平”**

------

### 但 AI 不擅长（也是你该重点提升的）：

#### 1️⃣ 需求理解 & 抽象

AI 不知道：

- 用户到底要什么
- 哪个需求是核心
- 哪些功能是“伪需求”

👉 你的价值：

- 把模糊需求 → 清晰系统设计

------

#### 2️⃣ 系统设计能力

AI 可以写代码，但不会真正设计系统：

比如：

- 表怎么设计？
- 权限怎么分层？
- 数据怎么流转？
- 接口怎么拆？

👉 你的价值：

- 决定系统结构（而不是写每一行代码）

------

#### 3️⃣ 技术选型能力

AI 不知道你的：

- 团队规模
- 业务复杂度
- 未来扩展性
- 成本约束

👉 你的价值：

- 为什么选 Next.js 而不是纯 React？
- 为什么用 Node 而不是 Python？
- 是否需要微服务？

------

#### 4️⃣ 代码“质量控制”

AI 代码的问题：

- 能跑 ≠ 可维护
- 能用 ≠ 可扩展
- 能生成 ≠ 没 bug

👉 你的价值：

- review AI 代码
- 重构
- 提升可读性
- 控制复杂度

------

#### 5️⃣ AI 结果校验（AI 时代新能力）

AI 会：

- 胡说
- 漏信息
- 给错逻辑

👉 你的价值：

- 校验结果
- 设计 fallback（兜底）
- 加规则约束

------

#### 6️⃣ 业务闭环能力（最重要）

AI 只能帮你写代码，但不会帮你：

- 把功能上线
- 接入用户
- 收集反馈
- 持续优化

👉 你的价值：
👉 **把一个功能从 idea → 上线 → 可用 → 可持续**

# 三、在 Cursor 时代，你应该怎么做？

重点来了👇

## ❌ 错误做法

很多人现在在做：

- 全靠 AI 写
- 不理解代码
- 一出 bug 就卡死
- 项目能跑但讲不清

👉 这种人未来最危险

------

## ✅ 正确做法（非常关键）

### 1️⃣ 把 AI 当“初级工程师”

不是当“答案机器”

你要：

- 给任务
- 拆需求
- 验收结果

------

### 2️⃣ 先设计，再让 AI 写

顺序必须是：

```text
你设计 → AI 实现 → 你 review → 你优化
```

而不是：

```text
AI 写 → 你复制 → 完了
```

------

### 3️⃣ 强化“读代码能力”

未来更重要的是：

👉 能快速看懂 AI 写的代码

------

### 4️⃣ 强化“改代码能力”

你要能：

- 重构 AI 代码
- 简化逻辑
- 提升性能

------

### 5️⃣ 强化“调试能力”

AI 时代最值钱能力之一：

👉 Debug

你要会：

- 看日志
- 定位问题
- 分析链路
