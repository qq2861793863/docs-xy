## 1.基于 OpenClaw

```markdown
# OpenClaw 配置 DeepSeek 模型简明教程

> 基于 OpenClaw 2026.3.28 (f9b1079) 实测，从零配置到使用 DeepSeek 官方 API。

---

## 1. 安装 OpenClaw

```bash
# 推荐使用 npm 全局安装（需 Node.js 18+）
npm install -g openclaw

# 或使用官方安装脚本
curl -fsSL https://openclaw.ai/install.sh | bash
```

验证安装：`openclaw --version`

---

## 2. 修复配置文件（如果遇到 `Unrecognized key: "provider"` 错误）

```bash
openclaw doctor --fix
```

> 该命令会自动修正旧版配置格式。

---

## 3. 启动 Gateway（首次运行）

```bash
openclaw gateway
```

保持终端运行，或按 `Ctrl+C` 停止（后续会安装为系统服务）。

---

## 4. 运行向导配置 DeepSeek 模型

```bash
openclaw onboard
```

### 向导选项速查（按顺序）

| 问题                                          | 选择                                    |
| --------------------------------------------- | --------------------------------------- |
| `I understand this is personal-by-default...` | `Yes`                                   |
| `Setup mode`                                  | `Manual`                                |
| `Config handling`                             | `Update values`                         |
| `What do you want to set up?`                 | `Local gateway (this machine)`          |
| `Workspace directory`                         | 默认回车                                |
| `Model/auth provider`                         | `DeepSeek (API key)`                    |
| `Enter DeepSeek API key`                      | 粘贴 `sk-...`                           |
| `Default model`                               | `Keep current (deepseek/deepseek-chat)` |
| `Gateway port`                                | 默认 `18789`                            |
| `Gateway bind`                                | `Loopback (127.0.0.1)`                  |
| `Gateway auth`                                | `Token (Recommended default)`           |
| `Tailscale exposure`                          | `Off (No Tailscale exposure)`           |
| `How to provide gateway token?`               | `Generate/store plaintext token`        |
| `Select a channel`                            | `Finished`                              |
| `Search provider`                             | `Skip for now`                          |
| `Configure skills now?`                       | `No`                                    |
| `Enable hooks?`                               | `Skip for now`                          |
| `Install Gateway service (recommended)`       | `Yes`                                   |

向导结束后会自动生成配置文件备份：`~/.openclaw/openclaw.json.bak`

---

## 5. 重启 Gateway 使配置生效

```bash
openclaw gateway restart
```

或如果安装为服务，重启电脑即可自动运行。

---

## 6. 测试 DeepSeek 是否正常工作

```bash
openclaw chat --message "你好，请介绍一下自己"
```

预期回复应包含“我是 DeepSeek，由深度求索创造”。

## 常用命令速查

| 用途                 | 命令                                   |
| -------------------- | -------------------------------------- |
| 查看配置状态         | `openclaw doctor`                      |
| 自动修复配置         | `openclaw doctor --fix`                |
| 启动 Gateway（前台） | `openclaw gateway`                     |
| 重启 Gateway         | `openclaw gateway restart`             |
| 停止 Gateway         | `openclaw gateway stop`                |
| 重新运行配置向导     | `openclaw onboard`                     |
| 修改单个配置项       | `openclaw config set <key> <value>`    |
| 查看当前默认模型     | `openclaw models status`               |
| 切换默认模型         | `openclaw models set <provider/model>` |
| 发送测试消息         | `openclaw chat --message "内容"`       |
| 卸载 Gateway 服务    | `openclaw gateway uninstall-service`   |

---

## 配置文件位置

- 主配置：`~/.openclaw/openclaw.json`
- 工作区：`~/.openclaw/workspace`
- 会话记录：`~/.openclaw/agents/main/sessions`
- 日志文件：`/tmp/openclaw/openclaw-*.log`（Linux/macOS）或 `%TEMP%\openclaw\`（Windows）

## 常见问题

### Q: `Unrecognized key: "provider"` 错误
**A:** 运行 `openclaw doctor --fix` 自动修复。

### Q: Gateway 启动后显示 `agent model: anthropic/claude-...`
**A:** 未成功切换到 DeepSeek，请重新运行 `openclaw onboard`，确保在 `Model/auth provider` 处选择 `DeepSeek (API key)`。

### Q: 如何更新 API Key？
**A:** 运行 `openclaw config set models.providers.deepseek.apiKey "新key"`，然后 `openclaw gateway restart`。

### Q: 想改回其他模型？
**A:** `openclaw models set openai/gpt-4`（示例），或重新运行向导。
