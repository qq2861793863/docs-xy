# 知识点汇总

这是一个非常棒的学习计划。Vue 3 和 React 是目前前端最主流的两大框架，掌握它们的核心逻辑对你的全栈能力（结合 PHP/Laravel）会有极大的提升。

考虑到你有后端背景（PHP/Laravel），你会发现 **Vue 3** 上手更顺滑（Laravel 社区官方偏爱 Vue），而 **React** 则更像是在写纯 JavaScript，灵活性极高。

以下我为你整理的 **核心知识点** 和 **最佳学习路线**，剥离了过时的内容，只保留现代开发必须掌握的重点。

------

### 第一部分：Vue 3 核心知识点 (重点：组合式 API)

Vue 3 最大的变革是引入了 **Composition API (组合式 API)**，配合 ``<script setup>`` 语法糖，代码极其简洁。

#### 1. 响应式系统 (Reactivity) **[重中之重]**

这是 Vue 的魔法来源，必须搞懂 `ref` 和 `reactive` 的区别。

- **`ref()`**: 用于定义基本类型（String, Number, Boolean）或单个对象。取值需要 `.value`。
- **`reactive()`**: 用于定义引用类型（Object, Array）。不需要 `.value`，但解构会丢失响应性（需配合 `toRefs`）。
- **`computed()`**: 计算属性，用于依赖其他响应式数据产生新数据，具有缓存特性。
- **`watch()` / `watchEffect()`**: 监听数据变化并执行副作用（如发请求）。

#### 2. 组件通信 (Component Communication)

- **Props (父传子)**: `defineProps`。
- **Emits (子传父)**: `defineEmits`。
- **Provide / Inject**: 跨级组件通信（依赖注入）。
- **Slots (插槽)**: 极其重要，用于封装高复用组件（如你在做后台管理时的弹窗、卡片）。

#### 3. 生命周期 (Lifecycle Hooks)

Vue 3 把生命周期钩子改成了函数导入式：

- `onMounted` (挂载后，发请求通常在这里)。
- `onUnmounted` (卸载时，清理定时器/事件监听)。

#### 4. 必备生态

- **Pinia**: **必学！** Vuex 已经过时了。Pinia 是新一代状态管理，API 极其简单，类似 React 的 Hooks。
- **Vue Router 4**: 路由管理，掌握动态路由匹配和导航守卫（做登录拦截用）。

------

### 第二部分：React 核心知识点 (重点：Hooks)

React 的核心思想是 **“UI = f(state)”**，一切皆 JS 函数。抛弃“类组件 (Class Component)”，只学“函数组件 (Functional Component)”。

#### 1. JSX 语法

- 在 JS 里直接写 HTML 标签。
- 注意：`class` 变成了 `className`，变量用 `{}` 包裹。

#### 2. Hooks (钩子函数) **[灵魂所在]**

React 没有 Vue 那样的“自动收集依赖”，你需要手动管理副作用。

- **`useState`**: 管理状态。`const [count, setCount] = useState(0)`。
- **`useEffect`**: 处理副作用（API 请求、DOM 操作）。**这是 React 最难也是最重要的点**，必须理解依赖数组 `[]` 的含义。
- **`useRef`**: 获取 DOM 元素或存储不触发渲染的变量。
- **`useMemo` / `useCallback`**: 性能优化，防止不必要的重新渲染。

#### 3. 组件通信

- **Props**: 单向数据流，父传子。
- **Context API**: 用于避免“Props Drilling”（层层传递），实现全局状态共享（如主题切换、用户信息）。

#### 4. 渲染控制

- **条件渲染**: 使用 `&&` 或三元运算符 `? :`。
- **列表渲染**: 使用 `.map()` 生成元素数组，必须加 `key`。

#### 5. 必备生态

- **React Router v6**: 路由。
- **Zustand** 或 **Redux Toolkit**: 状态管理。推荐先看 Zustand，极简，和 Vue 的 Pinia 体验很像。

------

### 第三部分：Vue 3 vs React 核心差异对比

| **特性**     | **Vue 3 (``<script setup>``)**                               | **React (Hooks)**                                            |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **核心思想** | **可变的 (Mutable)**：直接修改 `count.value++`，系统自动更新 UI。 | **不可变的 (Immutable)**：不能直接改，必须调用 `setCount(newVal)` 触发重绘。 |
| **心智模型** | **模板 + 逻辑**：HTML 和 JS 分离（虽然可以写在一起，但逻辑上是分开的）。 | **All in JS**：HTML 只是 JS 的语法糖，逻辑控制能力极强。     |
| **上手难度** | 低。API 语义化好（v-if, v-for）。                            | 中。需要较好的 JS 基础（闭包、数组操作）。                   |
| **适合场景** | 快速开发、中小型项目、后台管理系统、Laravel 项目。           | 超大型项目、跨平台 (React Native)、对灵活性要求极高的场景。  |

------

### 第四部分：推荐技术学习路线 (Roadmap)

考虑到你的全栈目标，我建议的路线如下：

#### 阶段一：JavaScript 基础补强 (1周)

*不要急着学框架，先把 ES6+ 搞定，否则看文档会很晕。*

- **箭头函数** `() => {}`
- **解构赋值** `const { data } = res`
- **展开运算符** `...args`
- **数组方法** `.map()`, `.filter()`, `.reduce()`
- **Promise / Async / Await** (处理后端 API 必备)

#### 阶段二：先攻 Vue 3 (2周)

*因为你熟悉 Laravel，Vue 会让你倍感亲切。*

1. 搭建 Vite + Vue 3 项目。
2. 学习 ``<script setup>`` 语法。
3. 实现一个简单的 **To-Do List**（增删改查）。
4. 接入 **Pinia** 管理状态。
5. 实战：用 **Element Plus** 或 **Ant Design Vue** 写一个后台管理的表格页面。

#### 阶段三：再看 React (2周)

*掌握了 Vue 的组件化思想后，学 React 主要是适应语法的不同。*

1. 搭建 Vite + React 项目。
2. 理解 `useState` 和 `useEffect`。
3. 重写上面的 To-Do List。
4. 感受 React 的“纯函数”思维。

#### 阶段四：进阶与全栈整合 (长期)

- **Next.js (React)** 或 **Nuxt.js (Vue)**: 这是现在的行业标准，支持 SSR (服务端渲染)，SEO 友好。
- **TypeScript**: 无论选 Vue 还是 React，最终都要上 TS。

### 💡 总结建议

如果你主要为了**自己做项目（如博客、工具站）\**或\**配合 PHP 开发**：**首选 Vue 3**。它的开发效率极高，且心智负担小。

如果你为了**找纯前端工作**或**想深入理解函数式编程**：**React 是必须学的**。

# 代码示例

没问题！代码是程序员通用的语言。我将针对上面提到的**核心知识点**，为你提供最直观的**“最小化代码片段”**。

这会帮你快速建立“Vue 这样写，React 那样写”的心理映射。

### Part 1: Vue 3 (使用 `<script setup>`)

Vue 3 的特点是：**响应式数据自动追踪**，代码看起来很像普通的 JS 变量赋值。

#### 1. 响应式系统 (`ref` vs `reactive`)

**场景：** 定义一个数字（基本类型）和一个用户信息（对象）。

```html
<script setup>
import { ref, reactive } from 'vue'

// 1. ref: 用于基本类型，必须通过 .value 修改
const count = ref(0)
const add = () => {
  count.value++ // 注意这里要加 .value
}

// 2. reactive: 用于对象，直接修改属性
const user = reactive({ name: '筱宇', role: 'Admin' })
const changeName = () => {
  user.name = 'Xiaoyu' // 不需要 .value，像普通对象一样操作
}
</script>

<template>
  <button @click="add">{{ count }}</button>
  <p>{{ user.name }}</p>
</template>

```

#### 2. 生命周期与监听 (`onMounted`, `watch`)

**场景：** 页面加载时请求 API，并监听搜索框的变化。

```javascript
<script setup>
import { ref, onMounted, watch } from 'vue'

const searchText = ref('')

// 生命周期：挂载完成后执行 (类似 jQuery 的 ready)
onMounted(() => {
  console.log('页面初始化完成，发送请求...')
})

// 监听：当 searchText 变化时触发
watch(searchText, (newValue, oldValue) => {
  console.log(`搜索词从 ${oldValue} 变为 ${newValue}`)
  // 这里可以防抖后发送搜索请求
})
</script>

```

#### 3. 组件通信 (父子组件)

**场景：** 父组件传标题给子组件，子组件通知父组件关闭。

```javascript
// --- 子组件 (Child.vue) ---
<script setup>
// 接收 Props
const props = defineProps({
  title: String
})

// 定义 Emits
const emit = defineEmits(['close'])

const closePanel = () => {
  emit('close', '关闭的数据payload') // 触发事件
}
</script>

<template>
  <div class="modal">
    <h1>{{ title }}</h1>
    <button @click="closePanel">关闭</button>
  </div>
</template>

// --- 父组件使用 ---
// <Child title="我的博客" @close="handleClose" />

```

### Part 2: React (使用 Hooks)

React 的特点是：**显式状态更新**，你必须告诉 React “我要改数据了，请重新渲染”。

#### 1. 状态管理 (`useState`)

**场景：** 同样的计数器。

```javascript
import { useState } from 'react';

function Counter() {
  // 定义状态：[当前值, 修改值的函数]
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      点击次数: {count}
    </button>
  );
}

```

#### 2. 副作用 (`useEffect`) **(React 最核心)**

**场景：** 模拟 Vue 的 `onMounted` 和 `watch`。`useEffect` 把这两个功能合并了。

```javascript
import { useEffect, useState } from 'react';

function UserProfile({ userId }) {
  const [data, setData] = useState(null);

  // 用法 1: 依赖数组为空 [] -> 等同于 onMounted (只执行一次)
  useEffect(() => {
    console.log('组件挂载了');
    
    // 返回一个函数 -> 等同于 onUnmounted (卸载时清理)
    return () => console.log('组件卸载了');
  }, []);

  // 用法 2: 依赖数组有值 [userId] -> 等同于 watch (userId 变了就执行)
  useEffect(() => {
    console.log('User ID 变了，重新获取数据...');
    // fetchData(userId)
  }, [userId]); 

  return <div>...</div>;
}

```

#### 3. 渲染控制 (Map & 条件)

**场景：** 渲染一个文章列表，如果没数据显示 loading。

```javascript
function ArticleList({ articles }) {
  return (
    <div>
      {/* 条件渲染：使用逻辑与 && 或 三元运算符 */}
      {articles.length === 0 ? (
        <p>Loading...</p>
      ) : (
        <ul>
          {/* 列表渲染：使用 map */}
          {articles.map(item => (
            <li key={item.id}>
              {item.title}
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}

```

### Part 3: 直观对比 (Vue vs React)

让我们用同一个**“输入框同步显示 (双向绑定)”**的功能来看看差异。

**Vue 3 写法 (v-model 魔法):**

```html
<script setup>
import { ref } from 'vue'
const text = ref('')
</script>

<template>
  <input v-model="text" />
  <p>你输入了: {{ text }}</p>
</template>

```

**React 写法 (手动控制):**

```javascript
import { useState } from 'react';

function InputBox() {
  const [text, setText] = useState('');

  return (
    <>
      {/* 必须手动监听 onChange 并调用 setText */}
      <input 
        value={text} 
        onChange={(e) => setText(e.target.value)} 
      />
      <p>你输入了: {text}</p>
    </>
  );
}

```

### 💡 总结与建议

* **Vue 的代码**通常更短，因为它帮你在底层做了很多“自动”的事情（比如 `v-model`）。
* **React 的代码**更直白，数据流向非常清晰（单向数据流），哪里变了需要你明确写出来。
