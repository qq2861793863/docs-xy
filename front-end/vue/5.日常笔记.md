# 日常笔记

## github 提交代码链接超时

注意修改成自己的 IP 和端口号

git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

查看代理

git config --global --get http.proxy
git config --global --get https.proxy

取消代理

git config --global --unset http.proxy
git config --global --unset https.proxy





## github新电脑提交代码

### 1. 生成新的 SSH 密钥
在你的 MINGW64 终端里直接输入这行（邮箱可以换成你自己的，不换也没关系）：
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```
*   **敲完回车后：** 它会问你 `Enter file in which to save the key`，**直接按回车**（用默认路径）。
*   **接着：** 它会问你 `Enter passphrase`，**还是直接按回车**（不设密码，省得以后烦）。
*   **再接着：** 再次按回车确认。

直到你看到一个方框字符画（Randomart image），说明密钥已经躺在 `~/.ssh/` 文件夹里了。

---

### 2. 查看并复制公钥
现在你再运行刚才那个报错的命令，就能看到内容了。输入以下命令把这一长串东西打印出来：
```bash
cat ~/.ssh/id_ed25519.pub
```
**操作：** 用鼠标选中屏幕上打印出来的以 `ssh-ed25519` 开头、以你邮箱结尾的那一整行，**右键点击复制**。

---

### 3. 把公钥交给 GitHub
1.  回到你的 GitHub 页面，点击右上角头像 -> **Settings**。
2.  左边菜单点 **SSH and GPG keys**。
3.  点击绿色按钮 **New SSH key**。
4.  **Title** 随便填（比如：`Desktop-Home`），**Key** 这一栏把刚才复制的内容粘进去。
5.  点击 **Add SSH key**。

---

### 最后测试一下：
回到终端，输入：
```bash
ssh -T git@github.com
```
如果你看到一段话里包含 **"successfully authenticated"**，那恭喜你，这条路彻底打通了！

**打通之后，你之前那个 `docs-xy` 文件夹里的 `git push` 应该就能秒过了。如果还提示权限问题，记得确认一下你的项目远程地址是不是已经改成了 `git@github.com:...` 这种 SSH 格式。**
